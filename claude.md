# Claude Project Context – Media Chrétien

## Objectif du projet
Ce projet est un **blog statique et éditorial** destiné à publier des **extraits courts du Message du prophète Kacou Philippe**.
Il ne remplace pas le site officiel, mais sert de média de diffusion simple, clair et respectueux.

Site officiel de référence :
https://www.philippekacou.org

---

## Stack technique
- Framework : **Next.js (App Router)**
- Contenu : **Markdown local + mdx-remote**
- UI : **Tailwind CSS + shadcn/ui**
- Base de données : **Aucune**
- Authentification : **Aucune**
- Déploiement : **Static export possible**
- Langage principal : **TypeScript**

---

## Contraintes importantes
- Le projet doit rester **simple, lisible et statique**
- Aucun système de login
- Aucun CMS externe
- Aucun backend dynamique
- Les articles sont stockés **localement en fichiers Markdown**
- Pas de sur-architecture inutile

---

## Ligne éditoriale
- Textes courts
- Ton sobre et respectueux
- Fidélité stricte au Message
- Aucune interprétation personnelle
- Aucune modification doctrinale

---

## Design & UI
- Couleurs principales :
  - Jaune
  - Vert
  - Blanc
- Variantes autorisées (teintes claires et foncées)
- UI épurée
- Utilisation exclusive de **shadcn/ui** pour les composants
- Pas d’effets visuels excessifs

---

## Pages attendues
- `/` : Accueil avec texte introductif + liste d’extraits
- `/articles/[slug]` : Lecture d’un extrait
- `/mentions` (optionnel) : Mention du site officiel
- Footer avec lien vers le site complet

---

## Règles pour Claude Code
- Toujours privilégier la **simplicité**
- Éviter les abstractions complexes
- Pas de dépendances inutiles
- Code lisible, maintenable, pédagogique
- Respecter les conventions Next.js App Router
- Ne pas proposer d’authentification ou de backend

---

## Modèles IA utilisés via OpenRouter
- Sonnet : Coding principal
- Opus : Architecture et réflexion
- Haiku : Ajustements rapides

---

## Ce que Claude doit éviter
- Ajouter des fonctionnalités non demandées
- Introduire une base de données
- Proposer un CMS
- Modifier le positionnement spirituel du projet
- Générer du contenu doctrinal inventé

---

## Priorité absolue
Clarté, fidélité, sobriété, respect du Message.
